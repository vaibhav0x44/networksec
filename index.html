Basic Definitions (Q1-30)

1. Define network security.

Network security refers to the policies, tools, and practices designed to protect network infrastructure, data, and services from unauthorized access, misuse, malfunction, modification, destruction, or improper disclosure.

  • Key objectives: Confidentiality, Integrity, Availability (CIA triad)
  • Implementation examples: Firewalls, encryption, intrusion detection systems
Memory Tip: Think of network security as a "digital immune system" protecting against cyber threats.

2. What are the three main goals of network security?

The CIA triad forms the core objectives:

  1. Confidentiality - Ensuring information is accessible only to authorized entities (e.g., encryption)
  2. Integrity - Maintaining accuracy and consistency of data (e.g., hashing)
  3. Availability - Guaranteeing reliable access to systems/resources (e.g., DDoS protection)
Memory Tip: "CIA" - Confidentiality keeps secrets, Integrity prevents tampering, Availability keeps services running.

3. Define authentication and authorization.

Authentication Authorization Examples
Verifies identity ("Are you who you claim to be?") Determines access rights ("What are you allowed to do?") Username/password → File permissions
Methods: Passwords, biometrics, tokens Methods: ACLs, RBAC, policies Fingerprint scan → Admin privileges
Memory Tip: Authentication = ID check at door, Authorization = what rooms you can enter.

4. What is confidentiality in network security?

Confidentiality ensures sensitive information is accessible only to authorized parties.

  • Implementation techniques:
    • Encryption (AES, RSA)
    • Access controls (RBAC)
    • Data masking
  • Example: HTTPS encrypts web communications

5. What do you mean by data integrity?

Data integrity guarantees information remains unaltered from its source to destination.

  • Protection methods:
    • Hash functions (SHA-256)
    • Digital signatures
    • Checksums
  • Example: File downloads with MD5 verification
Memory Tip: Integrity = "data fingerprint" that changes if tampered with.

6. Define anonymity in the context of network security.

Anonymity ensures users can perform actions without revealing their identity.

  • Techniques: Tor networks, proxy chains, cryptocurrency
  • Applications: Whistleblowing, privacy protection
  • Challenge: Balancing anonymity with accountability

7. What is a network security model?

A network security model is a formal framework describing security services and mechanisms.

  • Components:
    • Security policies
    • Threat assumptions
    • Defense mechanisms
  • Example: OSI Security Architecture (X.800)

8. List any two network security threats.

Common threats include:

  1. Malware (viruses, worms, ransomware)
  2. Phishing (social engineering attacks)
  3. DDoS attacks (flooding services with traffic)

9. What is the difference between SSL and TLS?

SSL (Secure Sockets Layer) TLS (Transport Layer Security)
Older protocol (SSL 3.0 deprecated in 2015) Modern replacement (TLS 1.3 current standard)
Vulnerable to POODLE attack Improved security algorithms
Uses MAC for integrity Uses HMAC for integrity
Memory Tip: TLS is the "upgraded SSL" - think "SSL → TLS" as "HTTP → HTTPS".

10. Define Public Key Infrastructure (PKI).

PKI is a framework for managing digital certificates and public-key encryption.

  • Components:
    • Certificate Authorities (CAs)
    • Registration Authorities
    • Certificate databases
  • Example: Verifying website authenticity via HTTPS

11. What is a digital signature?

A digital signature is a cryptographic technique that:

  • Verifies authenticity of digital messages/documents
  • Provides non-repudiation (sender cannot deny sending)
  • Created using sender's private key, verified with public key
  • Example: Signing PDF documents with Adobe Certificate
Memory Tip: Digital signature = electronic wax seal.

12. Name two types of firewalls.

Common firewall types:

  1. Packet-filtering firewalls (examine packet headers)
  2. Application-level gateways (proxy firewalls)
  3. Stateful inspection firewalls (track connection state)

13. What is the role of a firewall in network security?

Firewalls act as network traffic filters:

  • Primary functions:
    • Block unauthorized access
    • Allow legitimate traffic
    • Log network activity
  • Deployment: Between trusted/untrusted networks

14. Define intrusion detection system (IDS).

An IDS monitors networks/systems for malicious activity.

  • Types:
    • Network-based (NIDS)
    • Host-based (HIDS)
  • Detection methods: Signature-based, anomaly-based

15. What is a packet filtering firewall?

Packet-filtering firewalls:

  • Examine packet headers (IP addresses, ports, protocols)
  • Make allow/deny decisions based on rules
  • Advantage: Fast processing
  • Limitation: Cannot inspect payload content

16. Define a Virtual Private Network (VPN).

A VPN creates secure, encrypted connections over public networks.

  • Uses:
    • Remote access security
    • Bypassing geo-restrictions
    • Anonymous browsing
  • Protocols: IPsec, OpenVPN, WireGuard

17. What is Kerberos?

Kerberos is a network authentication protocol:

  • Uses ticket-granting system
  • Prevents eavesdropping/replay attacks
  • Common in Windows Active Directory
  • Components: KDC, TGS, Client, Service

18. What is the function of X.509 certificates?

X.509 certificates:

  • Bind identities to public keys
  • Used in SSL/TLS, S/MIME
  • Contain: Owner info, public key, CA signature
  • Example: Website SSL certificates

19. What is PGP (Pretty Good Privacy)?

PGP is an encryption program for:

  • Email/file encryption
  • Digital signatures
  • Uses hybrid cryptosystem (RSA + AES)
  • Key management: Web of trust model

20. Define S/MIME.

S/MIME (Secure/Multipurpose Internet Mail Extensions):

  • Standard for encrypted/signed email
  • Uses X.509 certificates
  • Common in enterprise email systems

21. What is IPsec?

IPsec (Internet Protocol Security):

  • Suite of protocols for secure IP communications
  • Provides: Confidentiality, integrity, authentication
  • Modes: Transport (host-to-host), Tunnel (network-to-network)

22. What are the two major protocols used in IPsec?

IPsec core protocols:

  1. AH (Authentication Header) - integrity/authentication
  2. ESP (Encapsulating Security Payload) - confidentiality/integrity

23. What is the SNMP architecture?

SNMP (Simple Network Management Protocol) architecture:

  • Components:
    • Managed devices
    • Agents
    • NMS (Network Management System)
  • Versions: SNMPv1/v2c (insecure), SNMPv3 (secure)

24. Define intrusion detection.

Intrusion detection is the process of:

  • Monitoring systems/networks for policy violations
  • Identifying malicious activity patterns
  • Generating alerts for suspicious events

25. What is the role of honeypots in network security?

Honeypots are decoy systems that:

  • Attract attackers away from real systems
  • Gather intelligence about attack methods
  • Types: Low-interaction (emulated), high-interaction (real systems)

26. Define distributed intrusion detection.

Distributed IDS (DIDS):

  • Monitors multiple network segments
  • Correlates events across systems
  • Examples: Prelude, OSSEC

27. What is the importance of network-based intrusion detection?

Network-based IDS (NIDS):

  • Monitors entire network segments
  • Detects attacks missed by host-based systems
  • Examples: Snort, Suricata

28. What is an intrusion detection exchange format?

IDMEF (Intrusion Detection Message Exchange Format):

  • Standardized alert format (RFC 4765)
  • Enables interoperability between security tools
  • Uses XML schema

29. Define host-based intrusion detection.

Host-based IDS (HIDS):

  • Monitors individual host systems
  • Tracks file changes, log files, processes
  • Examples: Tripwire, OSSEC

30. What is an adaptive intrusion detection system?

Adaptive IDS:

  • Learns normal behavior patterns
  • Adjusts detection thresholds dynamically
  • Uses machine learning techniques

32. Differentiate between symmetric and asymmetric encryption.

Feature Symmetric Encryption Asymmetric Encryption
Keys Single shared key Public/private key pair
Speed Faster (suitable for bulk data) Slower (used for key exchange)
Key Distribution Secure channel required Public keys freely distributed
Examples AES (128/256-bit), DES RSA (2048-bit), ECC
Memory Tip: Symmetric = 1 key (like a diary lock), Asymmetric = 2 keys (like mailbox - public=slot, private=key).

33. Explain how SSL/TLS ensures secure communication over the internet.

SSL/TLS provides security through:

  1. Handshake Protocol:
    • Negotiates cipher suite
    • Authenticates server (optionally client)
    • Establishes shared secret via asymmetric crypto
  2. Record Protocol:
    • Encrypts data with symmetric keys
    • Provides message integrity via HMAC
  3. Key Exchange:
    • RSA (legacy) or ECDHE (forward secrecy)
Memory Tip: SSL/TLS = Digital "handshake" (agree on keys) + "locked briefcase" (encrypt data).

34. Describe the working principle of a digital signature.

Digital signatures work via:

  1. Signing Process:
    • Sender hashes the message (SHA-256)
    • Encrypts hash with private key
    • Appends encrypted hash to message
  2. Verification Process:
    • Recipient decrypts signature with sender's public key
    • Computes fresh hash of received message
    • Compares two hashes for match
Memory Tip: Digital signature = Electronic wax seal (unique to sender, verifiable by all).

Memory Tip: VPN = Secure "tunnel" through the internet jungle.
